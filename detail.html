About

성장배경

군무원으로 40년간 한 자리를 지킨 아버지 밑에서 자랐습니다. "맡은 일은 끝까지 해라"가 집안의 규칙이었고, 그게 몸에 배었습니다. 고등학교 때 중학교까지 반 2등이던 성적을 1등으로 올리겠다고 정하고, 새벽 4시 등교·자정 하교를 3년간 반복해서 실제로 해냈습니다.

대학 졸업 후 펫 쇼핑몰 MD, 산림청 행정, 반도체 FAB 배관 도면 검토까지 분야가 다른 일을 거쳤습니다. 어디서든 엑셀을 만지고 데이터를 정리하는 시간이 가장 몰입이 됐고, 그게 적성 신호라는 걸 반도체 현장에서 확신했습니다. 2026년 2월 국비 6개월 과정에 들어가 개발을 시작했고, 첫 프로젝트에서 Supabase 인증을 혼자 붙여보다 새벽 3시까지 오류를 잡은 날, "이 일을 계속 하고 싶다"고 느꼈습니다.

본인의 장단점

장점은 처음 보는 기술도 직접 부딪혀 익히는 빠른 학습력과 끈기입니다. 6개월 과정에서 Firebase, Supabase, OpenAI API, Gemini API, Make 자동화 등 수업에서 다루지 않은 기술들을 공식 문서와 자료를 찾아가며 직접 프로젝트에 적용했습니다. 오류가 발생하면 원인을 분석하고 해결하는 과정을 반복했고, 그 결과 AI 기반 식단 추천, 실시간 데이터 연동, AI 리모델링 시뮬레이션 등 다양한 기능을 구현하며 3개의 프로젝트를 완성할 수 있었습니다.

팀 프로젝트에서는 공통 HTML 템플릿을 제작하거나 TypeScript 기반 틀을 배포한 뒤, 팀원들이 동일한 환경에서 개발할 수 있도록 기반을 마련했습니다. 코드 병합 과정의 CSS 충돌과 URL 오류를 해결하며 팀 전체가 원활하게 개발할 수 있도록 지원했고, Notion으로 수정 내역과 진행 상황을 공유하며 협업 효율을 높이는 역할도 맡았습니다.

단점은 처음 접하는 업무를 충분히 이해한 뒤 시작하려는 신중한 성격 탓에 초반 속도가 다소 느릴 수 있다는 점입니다. 이를 보완하기 위해 '먼저 작동하는 버전을 만들고 이후 개선한다'는 방식으로 개발 습관을 바꿔왔고, 팀장으로서 일정 내 결과물을 완성해야 하는 책임을 두 번 경험하며 이 부분이 많이 개선되었습니다.

업무 관련 역량

6개월 KDT 과정에서 팀 프로젝트 3개를 완성하며 서비스 기획부터 데이터베이스 설계, API 연동, AI 기능 구현, 배포까지 전 과정을 경험했습니다.

레디플레이 — 스포츠 입문자를 위한 큐레이션 커머스 웹사이트로, 3인 팀의 팀장을 맡아 공통 HTML 템플릿과 레이아웃을 제작해 팀원들이 동일한 환경에서 개발할 수 있도록 기반을 마련했습니다. 회원가입부터 결제까지의 전체 서비스 흐름을 구현했고, Supabase와 Make 자동화를 연동해 고객 문의 접수 시 Discord로 즉시 알림이 전송되는 CS 자동화 시스템을 구축했습니다.

먹어도 돼? — 4인 팀(프론트엔드 2명·백엔드 2명)에서 백엔드 팀장을 맡았습니다. Django와 Firebase 기반의 AI 식품 안전 정보 서비스로, Gemini API를 연동하고 사용자 정보를 프롬프트에 반영한 개인 맞춤형 식단 추천 기능을 설계했습니다. API 응답 오류 시 자동 재시도 로직과 24시간 캐싱을 구현해 서비스 안정성을 높였고, Celery/Redis 기반 주간 식단 추천 기능도 완성했습니다.

깨비터 — 4인 팀에서 충청북도 공공데이터 915행을 Supabase에 적재하고, React Native와 TypeScript, OpenAI API를 활용해 빈집 매칭 및 AI 리모델링 시뮬레이션 앱을 개발했습니다. OAuth 소셜 로그인, 매물 조회, 견적 관리, Google Maps API 연동 기능을 구현하고 원스토어 출시를 준비했습니다.

이론 역량으로는 정보처리기사·정보처리산업기사·프로그래밍기능사·컴활 2급 등 4종의 필기 시험에 합격하며 운영체제, 데이터베이스, 네트워크, 보안 전반의 기초를 갖추었습니다.

지원동기

중학생 시절 포토샵을 독학으로 익힌 이후 컴퓨터로 무언가를 직접 만드는 일이 가장 좋았습니다. 개발을 배우며 그 감각은 더욱 선명해졌습니다. 내가 작성한 코드가 실제 서비스에서 동작하고, 사용자에게 가치를 제공하는 순간의 성취감이 다른 어떤 경험과도 달랐습니다.

6개월간 3개의 프로젝트를 완성하며 서비스 기획부터 AI 기능 구현까지 다양한 영역을 경험했습니다. 익숙하지 않은 기술이더라도 필요한 기능을 구현하기 위해 직접 찾아보고 적용하는 과정에서 개발자로 성장하고 있다는 것을 느꼈고, 사용자가 실제로 쓰는 서비스를 만드는 팀의 일원이 되고 싶다는 목표가 확고해졌습니다.

입사 후 포부

입사 초기에는 개발 환경과 팀의 업무 프로세스를 빠르게 파악하고, 작은 기능 하나라도 책임감 있게 완성하는 데 집중하겠습니다. 코드 리뷰와 피드백을 적극적으로 받아들이며, 6개월간 3개의 프로젝트를 완성하며 쌓은 '모르면 직접 찾아 해결하는 습관'으로 새로운 환경에서도 빠르게 적응하겠습니다.

중장기적으로는 단순히 주어진 기능을 구현하는 것을 넘어, 사용자 입장에서 서비스를 고민하고 더 나은 방향을 제안할 수 있는 개발자가 되고 싶습니다. 프론트엔드와 백엔드를 모두 이해하는 개발자로 성장해 서비스 전반을 파악하고, AI와 데이터 기반 기술을 적극적으로 활용하는 것이 목표입니다.
