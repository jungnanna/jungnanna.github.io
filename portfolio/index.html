01

깨비터

빈집 매칭 & AI 리모델링 시뮬레이션 앱

기간2026.05 – 2026.06 구성팀 프로젝트 역할기획 · 백엔드 · AI 기능 구현
사진 준비중

지방 빈집 문제 해결을 위한 빈집 매칭 서비스입니다. 공공데이터 기반으로 빈집 매물을 탐색하고, AI가 리모델링 후 모습을 시뮬레이션해줍니다. 2026 충청북도 공공데이터 활용 창업 경진대회 출품작입니다.

주요 기능

  • 공공데이터 기반 빈집 매물 탐색 — 흩어져 있던 CSV 데이터를 정제해 915건 규모의 매물 DB 구축
  • AI 리모델링 시뮬레이션 — 빈집 사진을 분석하고 리모델링된 모습을 이미지로 생성
  • Naver / Google 소셜 로그인

What I did

  • AI 리모델링 기능 구현 — 이미지 분석과 생성형 이미지 API 연동, 결과 이미지 Supabase Storage 저장
  • 공공데이터 CSV 정제 및 Supabase(PostgreSQL) 매물 테이블 설계·구축
  • Naver / Google 소셜 로그인 연동 및 인증 흐름 디버깅

Trouble Shooting

AI API 호출 시 할당량 초과(429) 오류가 반복 발생

→ 일시적 오류와 할당량 소진을 구분해 원인을 나눔. 기능별 API 키 분리로 장애를 격리하고, 재시도 로직을 추가해 안정화

먹어도 돼?

개인 맞춤 식단 추천 & 영양 분석 웹 서비스

기간2026.04 구성팀 프로젝트 역할백엔드 팀장
사진 준비중

개인별 건강 정보와 알레르기를 반영한 맞춤형 식단 추천·영양 분석 서비스입니다. AI 챗봇과 매주 자동 생성되는 AI 주간 식단 계획을 제공합니다.

주요 기능

  • 개인 맞춤 식단 추천 및 영양 분석 — 알레르기 정보 반영
  • AI 챗봇 — 식품 관련 질문에 실시간 응답
  • AI 주간 식단 계획 — Celery + Redis로 매주 자동 생성 스케줄링

What I did

  • Firebase / SQLite 이중 저장 구조 설계 — 서비스 데이터와 인증 데이터의 역할 분리
  • AI 프롬프트 개인화 설계 — 사용자 건강 정보를 반영한 프롬프트 구성
  • 백엔드 팀장으로서 협업 구조, 문서화, 오류 대응 방향 결정

Trouble Shooting

Gemini API 503(서버 과부하) 오류로 챗봇 응답 실패

→ 일시 오류로 판단, 재시도 로직을 넣어 응답 성공률 개선

챗봇과 식단 생성이 같은 API 키를 써서 한쪽 할당량 소진 시 전체 기능 중단

→ 기능별 API 키 분리로 장애 범위를 격리, 한 기능이 죽어도 다른 기능은 유지

레디플레이

스포츠 입문자를 위한 큐레이션 커머스 웹사이트

기간2026.03 구성3인 팀 역할팀장 · 공통 구조 설계
사진 준비중

스포츠 입문자가 종목별 필수 장비를 쉽게 고를 수 있도록 큐레이션한 커머스 웹사이트입니다. 프레임워크 없이 HTML/CSS/JS로 구현했습니다.

주요 기능

  • 종목별 큐레이션 상품 탐색 및 장바구니
  • 로그인 / 장바구니 상태 유지 — Session Storage 활용
  • Toss Payments 결제 연동
  • Discord 자동 알림 CS 시스템 — 문의 발생 시 팀 채널로 실시간 전송

What I did

  • fetch API 기반 공통 헤더·푸터 구조 설계 — 페이지마다 중복되던 코드를 한 곳에서 관리
  • Session Storage 기반 로그인/장바구니 상태 유지 구현
  • Discord Webhook 연동 CS 알림 시스템 구축
  • 3인 팀 팀장으로서 역할 분배와 일정 관리

Trouble Shooting

페이지마다 헤더·푸터 코드가 중복되어 수정할 때마다 모든 파일을 고쳐야 하는 문제

→ fetch API로 공통 HTML을 불러오는 구조로 변경, 한 파일만 고치면 전체 반영되도록 개선
